     - **O40 `blind-spot-pass`** — When work enters territory the user signals — or you infer — is unfamiliar to them, run a blind-spot pass before pinning outcomes: delegate exploration to surface the unknown unknowns, then teach the user enough to react. Why: the interview (O18) only drains unknowns someone already perceives; a user who doesn't know what good looks like can't accept criteria for it. Homes: B§clarifying-outcomes.
     - **O41 `user-comprehension-gate`** — A two-ended contract around autonomous runs. **At dispatch/plan time**: set the autonomy remit with the user — how autonomous this run is, which decision classes the main agent may settle on best judgement, which must be surfaced. **At the ratification/merge point**: offer an explainer + comprehension check (quiz) covering the *unratified decision risk* the run accumulated — divergence from the ratified plan, consequential/hard-to-reverse decisions made mid-flight that got less exploration than their stakes warranted, and anything that fell outside or strained the agreed remit. Why: ratification without comprehension is rubber-stamping — the user-facing kin of R4 `distrust-frictionless-approval` — and the remit is what makes the trigger computable rather than a vibe. Calibrations (load-bearing, keep all three): an **offer** the human takes or declines, never an agent-enforced gate; the trigger is **unratified decision risk, never volume** — raw diff size, run length, or "the user wasn't watching" must not fire it (that would tax autonomy itself; a long run that held the plan with all decisions pre-ratified earns no quiz); the gate lives **at the ratification point only** — it never pauses or interrupts autonomous execution. Homes: B§execution (gate) — the remit-setting sentence may sit in B§clarifying-outcomes or B§planning if it reads more naturally there (same item, two ends; implementer's call, cite O41 from both spots). Maintainer-ratified revision 2026-07-07 (superseding the earlier delta-vs-attention draft — record the supersession in task 18's decision node, not here).
