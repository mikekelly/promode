# Mine marketingskills into candidate MK register opinions

## Brief
- **Orient** — Source: https://github.com/coreyhaines31/marketingskills (MIT, Corey Haines; 47 skills, each `SKILL.md` + `references/` + `evals/`). A prior inventory pass (2026-07-15) classified **11 skills as opinion-dense/mixed**: `ads`, `offers`, `directory-submissions`, `marketing-loops`, `marketing-council`, `ad-creative`, `copy-editing`, `cold-email`, `customer-research`, `launch`, `marketing-plan`, `ai-seo` — plus 4 TUTORIAL-classified skills whose payload sits unread in `references/`: `pricing`, `revops`, `marketing-psychology`, `marketing-ideas`. Register format: `plugins/promode/docs/opinion-register.md` (slug + one-line statement + homes; read its preamble + one section for the house style).
- **Specify** — deliver `tasks/30-mk-opinion-candidates.md`: a candidate list for the maintainer's **item-by-item ratification**. Per candidate: proposed slug (`mk-*`), one-line statement, source (skill + tight quote/paraphrase), **tier**, perishability flag, proposed homes (CMO def / senior-marketer def / routed doc / brief), and its why. The three tiers (ratified 2026-07-15) govern treatment:
  1. **Name-only** — framework the pinned model (inherit/opus) reliably knows: the opinion wraps the bare name (the when/which/why); never spell out, never name-drop for coverage.
  2. **Routed-doc** — named framework the model only half-knows (e.g. Seven Sweeps, ORB, grounded-inputs): def carries the decision + doc pointer; mechanics spelled out once in a routed doc. When unsure of tier 1 vs 2, probe the pinned model cold before deciding.
  3. **Spelled-out + currency** — claims contradicting model priors or postdating training (statics>video, net-cash>ROAS, creative-is-the-targeting): full statement + why + verified-as-of date + re-verify trigger (⚙-analogue).
  Close with a **rejected-material list**: what was deliberately not mined, one line why each (e.g. "model-derivable", "perishable tool table", "compliance detail better re-derived live").
- **Why** — the register is the owner's preferences (M0): Corey's doctrine is raw material, not a donor of defaults — candidates exist so the maintainer ratifies each as *his*. This doc gates the whole marketing lane (tasks 31–35).
- **Verified vs assumed** — the inventory's classifications were made at SKILL.md depth; `references/` subfiles were NOT read — read them for the 11 + spot-check the 4 named above (a TUTORIAL call may flip one level down). Quoted benchmarks (e.g. "6-27× higher") are the repo's claims, unverified — mining ≠ endorsing; every tier-3 candidate must carry its unverified status explicitly.
- **Not / exit** — do NOT edit the register, any def, the brief, or README; do NOT fabricate evidence grades or verify benchmarks yourself. Exit: candidates doc committed + Outcome recorded here.
