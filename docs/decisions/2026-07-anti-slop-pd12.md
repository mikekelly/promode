## What was decided

- **PD12 `decisions-not-defaults`** — every element of a shipped visual/copy artifact is a decision someone can defend; AI slop is the *absence* of one — a machine default nobody chose. Framed as PD2's generative twin (PD2 removes the decisions users shouldn't have to make; PD12 demands the maker actually made the ones that ship). **Unchosen-minimalism clause:** stripping to the model's tasteful-safe house style is the newest default, not a decision.
- **SPD** carries the `<decisions-not-defaults>` authoring block and the red flag *"defended as 'that's what it generated'" — generation is not a decision*. **`VOCABULARY.md` owns the dated, per-project banned machine-voice patterns** (copy tells ride T29 — no new register row); tells drift with model generations, so the list is dated.
- **Auditor** refcon-check gains items 5–6, including *"are the references themselves decided?"* — the gate replays whatever the reference contains, so an undecided reference crystallises slop and enforces it forever (a slop element inside a reference outranks the same element in an implementation).
- **Refcon `frontend-design` boundary rewritten** around decided-vs-default triage, with the kill-ai-slop catalog cited in **exactly one live home** (the refcon doc; time-bound orientation, dated) — no tell-catalog text in any def (M1/K5).
- **Projects crystallise their own dated grep tell-scan — detection-only, never auto-fix** (an auto-fixer swaps one undecided output for another).
- **No brief edit** — verified in the original session: PD9's consult routing already covers when SPD engages.
- **Scope guard — memorability stays parked:** PD12 asks *"who chose this?"*, never *"is this memorable?"* (the perception-gap/memorability check remains a parked round-1 follow-up, not part of this doctrine).

## Decision log — rejected alternatives (durable reasons; don't re-suggest)

1. **Vendoring the kill-ai-slop catalog** into the plugin. Rejected: the repo is unlicensed — read, never vendor; and a vendored copy would freeze a drifting, time-bound tell list into doctrine.
2. **Shipping the discipline as a skill.** Rejected: promode ships no skills — voluntary invocation is non-determinate (M5, `2026-07-skills-elimination.md`); the discipline lives on non-voluntary surfaces (SPD def block, auditor items, routed-doc boundary).
3. **A standing "de-slop pass" agent.** Rejected: it would institutionalise *post-hoc scrubbing* when the whole point is that slop never enters the reference — decide upstream, don't launder downstream.
4. **Relying on the `frontend-design` skill alone.** Rejected: it generates aesthetic direction but enforces no provenance — its output is exactly the candidate material the decided-vs-default triage exists to gate.
5. **Recommending the upstream scanner** (kill-ai-slop's own tooling). Rejected: no license, two days old at ratification, Tailwind-specific — projects crystallise their own dated grep tell-scan instead.
6. **Adding a brief-level routing clause for anti-slop work.** Rejected: verified in the original session that PD9's existing consult routing (user-facing/UI changes → SPD) already delivers the discipline; a new brief clause would spend cap budget restating a dispatch that already happens. *(Recast from the recorded "No brief edit (PD9 routing already covers)".)*
7. **An auto-fix tell-scan** (scan and rewrite machine-voice copy). Rejected: detection-only is the ratified stance — an auto-fixer swaps one undecided output for another undecided output; the fix is a person (or agent) *deciding*, recorded in `VOCABULARY.md`. *(Recast from the recorded "detection-only, never auto-fix" stance.)*
